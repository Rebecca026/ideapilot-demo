# AI 产品经理全栈 Demo 上线学习路线

目标：从只会 vibe coding，走到能独立把一个 AI 工具做成公网可访问 demo。

## 总原则

- 先会跑通，再追求优雅。
- 先用托管平台，再学传统云服务器。
- 每一阶段都要产出一个可验证结果，而不是只看教程。

## 第一阶段：Netlify / Vercel + Supabase

你要学会把一个工具变成公网产品 demo。

### 你需要理解

- Netlify / Vercel：负责把网页和后端接口部署到公网。
- Supabase：负责保存数据，类似一个帮你管好的数据库。
- 前端：用户看到的页面。
- 后端：前端请求的接口，负责处理逻辑和保护密钥。
- 数据库：保存用户输入、生成结果、反馈和历史记录。

### 你要能完成

- 做一个输入框和按钮。
- 点击按钮后请求后端接口。
- 后端返回结果。
- 后端把这次输入保存到 Supabase。
- 打开公网链接，别人也能完成同样流程。

### 阶段通过标准

- 有一个 Netlify 或 Vercel 公网链接。
- 提交表单后页面能返回结果。
- Supabase 表里能看到新增数据。

## 第二阶段：GitHub 自动部署

你要学会像开发一样管理代码，而不是每次手动拖 zip。

### 你需要理解

- GitHub：放代码的地方。
- Commit：一次代码改动记录。
- Push：把本地代码同步到 GitHub。
- 自动部署：GitHub 一更新，Netlify / Vercel 自动重新上线。

### 你要能完成

- 建一个 GitHub 仓库。
- 把项目代码传上去。
- 连接 Netlify 或 Vercel。
- 改一行文案后 push，网站自动更新。

### 阶段通过标准

- 不再靠拖 zip 发布。
- 每次修改代码后，平台自动部署新版本。

## 第三阶段：API Key、环境变量、日志、数据库表

你要学会真实 AI 产品最容易踩坑的部分。

### 你需要理解

- API Key：调用 AI 或第三方服务的钥匙。
- 环境变量：上线平台保存秘密配置的地方。
- 日志：接口运行时留下的记录，用来排查问题。
- 数据表设计：决定你要保存什么数据，以及以后怎么分析。

### 你要能完成

- 把 OpenAI API Key 放到环境变量。
- 后端调用 AI API。
- 前端不暴露任何密钥。
- 出错时能去平台日志里找到原因。
- 能设计一张简单的数据表。

### 阶段通过标准

- AI 结果来自真实 API。
- 密钥没有写在前端代码里。
- 出错时知道去哪里看日志。
- Supabase 表里能保存 AI 生成结果和用户输入。

## 第四阶段：云服务器、Docker、域名、备案

这阶段更像正式工程部署，不是做 demo 的第一优先级。

### 你需要理解

- 云服务器：你自己租一台远程电脑。
- Docker：把应用和运行环境打包。
- 域名：把复杂链接变成自己的网址。
- 备案：中国大陆服务器绑定域名通常需要备案。

### 你要能完成

- 买一台云服务器。
- 用 Docker 跑一个后端服务。
- 配置域名。
- 理解什么时候需要备案。

### 阶段通过标准

- 能解释什么时候用 Vercel/Netlify，什么时候才需要阿里云这类服务器。

## 推荐学习顺序

1. 先用 Netlify + Supabase 跑通一个全栈 demo。
2. 再用 GitHub 管代码和自动部署。
3. 再接真实 AI API，学环境变量和日志。
4. 最后再碰云服务器和 Docker。
